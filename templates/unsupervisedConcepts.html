Module 03 · Theory

Basic Concepts

Foundations of unsupervised learning, clustering, and the K-Means algorithm

01 — What is Unsupervised Learning?

Unsupervised learning is a type of machine learning where the model works with unlabeled data — there are no predefined categories or correct output values. The algorithm must discover hidden patterns, structures, or relationships on its own.

Unlike supervised learning, where the model is guided by labeled examples, unsupervised learning lets the data speak for itself. This makes it especially useful when labeled data is scarce, expensive to obtain, or simply unavailable.

  • No labels required: the model learns directly from raw input data.
  • Pattern discovery: finds natural groupings, anomalies, or structure in the data.
  • Exploratory: often used as a first step before applying more complex models.

02 — What is Clustering?

Clustering is one of the main tasks in unsupervised learning. It consists of grouping data points based on similarity, so that elements within the same group share more characteristics with each other than with those in other groups.

The algorithm does not know what the groups represent — it only knows that some points are more similar to each other. The interpretation of what each cluster means is left to the analyst.

  • Intra-cluster similarity: points within the same cluster should be close to each other.
  • Inter-cluster separation: points in different clusters should be far apart.
  • No ground truth: cluster quality is measured by compactness, not by a correct label.

03 — What is K-Means?

K-Means is the most widely used clustering algorithm. It partitions a dataset into K groups by assigning each data point to the nearest cluster center, then updating those centers based on the new assignments. This process repeats until the solution stabilizes.

The value of K — the number of clusters — must be defined before running the algorithm. Choosing K correctly is one of the most important decisions in the process.

04 — Centroids

A centroid is the central point of a cluster — the geometric average of all data points assigned to that cluster. It represents the "center of gravity" of the group.

Centroids are recalculated after each assignment step. If a cluster contains points at coordinates (x₁,y₁), (x₂,y₂) … (xₙ,yₙ), its new centroid is:

μ = ( Σxᵢ / n , Σyᵢ / n )

μ — new centroid xᵢ, yᵢ — coordinates of each point n — number of points in the cluster

05 — Distance Metrics — Euclidean Distance

To assign each point to the nearest cluster, K-Means needs to measure distance. The most common metric is Euclidean distance — the straight-line distance between two points in space, derived from the Pythagorean theorem.

d = √( (x₂−x₁)² + (y₂−y₁)² )

d — distance between two points (x₁,y₁) — data point coordinates (x₂,y₂) — centroid coordinates

Each data point is assigned to the cluster whose centroid produces the smallest Euclidean distance. The process is applied to every point in every iteration.

06 — Iterative Process

01
Initialize centroids Choose K initial centroids — randomly or manually selected from the dataset.
02
Calculate distances Compute the Euclidean distance from each data point to every centroid.
03
Assign clusters Assign each point to the cluster with the nearest centroid.
04
Update centroids Recalculate each centroid as the mean of all points assigned to that cluster.
05
Repeat until convergence Go back to step 2. Stop when centroids no longer move significantly or assignments stop changing.

07 — Variance and Convergence

SSE — Sum of Squared Errors

The quality of clustering is measured by the SSE (inertia) — the sum of squared distances between each data point and its assigned centroid. A lower SSE means more compact, better-defined clusters.

SSE = Σₖ Σᵢ∈Cₖ ‖xᵢ − μₖ‖²

K — number of clusters xᵢ — data point μₖ — centroid of cluster k ‖·‖² — squared distance

Convergence is reached when the centroids stop moving between iterations — meaning the SSE has stabilized and further iterations would not improve the clustering. At this point the algorithm terminates.

  • SSE always decreases or stays the same between iterations.
  • A smooth, flattening SSE curve signals that the model is near convergence.
  • Comparing SSE across iterations shows how much the clustering improved at each step.

08 — Real-World Applications

Customer segmentation

Group customers by purchase behavior, age, or income to personalize marketing.

Document clustering

Organize news articles or research papers into topics without predefined categories.

Anomaly detection

Identify data points that do not belong to any cluster — potential fraud or system failures.

Image compression

Reduce image colors by grouping similar pixel values into K representative colors.

Genomics

Cluster gene expression patterns to identify groups of genes with similar behavior.

Geographic analysis

Group locations by proximity to optimize delivery routes or identify service zones.
